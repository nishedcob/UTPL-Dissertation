Titulación de Sistemas Informáticos y Computación

Desarrollo e implementación de una herramienta REPL Online para las plataformas educativas de la UTPL



Autor: Nicholas Spalding Earley-Dolenc

Director: Dr. Jorge López

Problema

Enseñar estudiantes a programar no es como otras materias de solo memorizacion, si no se tiene que orientar a enseñar estudiantes a pensar en como pueden resolver problemas desde el punto de vista de la computadora.

Actualmente en la Universidad Tecnica Particular de Loja, se enseña materias programacion de una forma clasica donde estudiantes son responsables por mantener su ambiente de aprendizaje de programacion y professores son responsables por la calificacion de codigo.

Estos actividades addicionales por parte de los estudiantes y sus professores resultan en tiempo perdido por ambos partes, tiempo que podria dedicar mejor al enseñanza y aprendizaje de la materia.

Solucion

Un plataforma en linea que combina un editor de codigo, control de versiones y un ambiente de ejecuccion seguro de codigo tanto para estudiantes con el fin de probar su codigo como sus professores y el sistema para revisar y calificar el mismo codigo.

Para realizar el mismo, se opto por una arquitectura orientada a servicios que permite extensibilidad y escalabilidad a futuro. Los servicios incluidos en el prototipo son:

  • GitEDU - Editor de Codigo en Linea
  • EduNube - Ejecutor de Codigo en Linea
  • GitServerHTTPEndpoint - Persistencia de Codigo en Linea

Tecnologias utilizados en la solucion (1 de 2):

  • En general
    • Twitter Bootstrap
    • Python 3 / Django
    • PostgreSQL
    • MongoDB
  • GitEDU - Editor de Codigo en Linea
    • jQuery
    • Ace Code Editor

Tecnologias utilizados en la solucion (2 de 2):

  • EduNube - Ejecutor de Codigo en Linea
    • Kubernetes / Docker
  • GitServerHTTPEndpoint - Persistencia de Codigo en Linea
    • Git / GitWeb

Resultados

  • El prototipo de GitEDU ofrece functionalidades LTI para integrar con sistemas de la Universidad a futuro como el EVA, ademas ofrece el interfaz necesario para el manejo de proyectos y codigo versionado en la plataforma por parte de estudiantes.
  • El prototipo de EduNube ofrece un API para la ejecuccion segura de codigo de forma escalable y es extensible para ofrecer mayor funcionalidad a futuro como se evoluciona las necesidades de la Universidad.
  • El prototipo de GitServerHTTPEndpoint ofrece un API para la persistencia de codigo bajo sistemas de control de versiones externos con la finalidad de que se puede compartir y revisar codigo en un formato estandarizado con usuarios y sistemas externos.

Conclusiones

Aunque se requiere mayor trabajo a futuro, en base al investigacion de trabajos similares, un plataforma para desarrollar, probar y calificar codigo en linea podria mejorar la calidad de enseñanza, evalucacion y aprendizaje con respeto a materias de programacion en la Universidad Tecnica Particular de Loja.
