Universidad Técnica Particular de Loja

Desarrollo e implementación de una herramienta REPL Online para las plataformas educativas de la UTPL


Autor: Nicholas Spalding Earley-Dolenc

Director: Dr. Jorge López

Titulación de Sistemas Informáticos y Computación

Problema

Enseñar a los estudiantes a programar no es como otras materias de memorización. Se debe orientarlos a pensar en como pueden resolver problemas desde el punto de vista de una computadora.

Actualmente en la Universidad Técnica Particular de Loja, se enseña materias de programación de una forma clásica donde los estudiantes son responsables por mantener su ambiente de aprendizaje de programación y los profesores son responsables por la calificación de código.

Estas actividades adicionales por parte de los estudiantes y de sus profesores resultan en tiempo perdido, tiempo que se podría dedicar a la enseñanza y aprendizaje de la materia.

Objetivos

Mejorar la enseñanza y evaluación de estudiantes de programación de la Universidad Técnica Particular de Loja.

Objetivos Específicos:

  • Integrar autenticación de los estudiantes con los LMS institucionales a través del protocolo LTI.
  • Proveer la capacidad de poder ejecutar código en línea.
  • Utilizar servidores de sistemas de control de versiones para persistir el código creado por estudiantes de forma externa.

Estado del Arte

Algunos Trabajos Relacionados

  • GitEduERP - Un editor de código prototipo en línea con persistencia de código con MongoDB y Snippets de GitLab.
  • Xen Web-based Terminal for Learning - Un sistema con máquinas virtuales de Xen para ofrecer un ambiente seguro para enseñar administración de sistemas Linux.

Algunos Sistemas Relacionados

  • Repl.it - Una plataforma educativa de alto costo con editor de código en línea, integración con sistemas educativos y calificación automatizada.
  • io.livecode.ch - Una plataforma prototipo abierta para enseñar programación, pero no maneja ni estados ni calificaciones.
  • Cloud9 IDE - Una plataforma para desarrollar y probar código en línea, no lleva integración educativa.

Solución

Un plataforma en línea que combina un editor de código, control de versiones y un ambiente de ejecución seguro de código tanto para estudiantes con el fin de probar su código como para sus profesores y el sistema para revisar y calificar el mismo código.

Para realizar el mismo, se optó por una arquitectura orientada a servicios que permite extensibilidad y escalabilidad a futuro. Los servicios incluidos en el prototipo son:

  • GitEDU - Editor de Código en Línea
  • EduNube - Ejecutor de Código en Línea
  • GitServerHTTPEndpoint - Persistencia de Código en Línea

Tecnologías utilizadas en la solución (1 de 2):

  • En general
    • Twitter Bootstrap
    • Python 3 / Django
    • PostgreSQL
    • MongoDB
  • GitEDU - Editor de Código en Línea
    • jQuery
    • Ace Code Editor

Tecnologías utilizadas en la solución (2 de 2):

  • EduNube - Ejecutor de Código en Línea
    • Kubernetes / Docker
  • GitServerHTTPEndpoint - Persistencia de Código en Línea
    • Git / GitWeb

Resultados

  • El prototipo de GitEDU ofrece funcionalidades LTI para integrarlo con sistemas de la Universidad a futuro como el EVA, además ofrece la interfaz necesaria para el manejo de proyectos y código versionado en la plataforma por parte de estudiantes.
  • El prototipo de EduNube ofrece un API para la ejecución segura de código de forma escalable y es extensible para ofrecer mayor funcionalidad a futuro como se evoluciona las necesidades de la Universidad.
  • El prototipo de GitServerHTTPEndpoint ofrece un API para la persistencia de código bajo sistemas de control de versiones externos con la finalidad de que se pueda compartir y revisar código en un formato estandarizado con usuarios y sistemas externos.

Conclusiones

Aunque se requiere mayor trabajo a futuro, en base al investigación de trabajos similares, una plataforma para desarrollar, probar y calificar código en línea podría mejorar la calidad de enseñanza, evaluación y aprendizaje con respeto a materias de programación en la Universidad Técnica Particular de Loja.

  • LTI es un estándar útil para tomar en cuenta en la implementación de soluciones con un fin educativo.
  • La combinación de Kubernetes con Docker proporciona una caja de herramientas muy potente para proveer soluciones de mayor complejidad, pero a su vez sistemas de producción confiables, escalables y portables, tanto en contextos como servicios como trabajos por lotes.
  • Los servidores de Git son altamente pesados, algo que se debe tomar en cuenta al momento de incluirlos en una solución o cuando se pide infraestructura para un proyecto.
  • Los bases de datos no relacionales permiten mayor libertad de persistir datos poco estructurados y simplifica la lógica de negocios que los mismos datos involucran.
  • La apariencia que una aplicación ofrece a sus usuarios finales es importante para mantener su atención dentro de cualquier sistema.
  • Django y Python orientado a objetos permite desarrollar con mayor rapidez, seguir buenas practicas de no repetir código y da mayor paso a extensibilidad a futuro.
  • Python 3 con las siguientes librerías:
    • Django
    • PyMODM
    • Bcrypt
    • PyJWT
    • Requests
    • ipython
    Forman una potente combinación para promover el desarrollo rápido de servicios de producción, siempre y cuando se maneje de forma adecuada el diseño de los mismos y el manejo de distintos tipos de datos.

Recomendaciones

  • Donde haya posibilidad, se recomienda reutilizar implementaciones de LTI para prevenir problemas de integración.
  • Se recomienda reutilizar trabajos de terceros para la creación de interfaces para poder dar mayor enfoque en la funcionalidad del backend.
  • Se recomienda siempre utilizar un ORM, tanto con bases de datos relacionales como los no relacionales ya que los mismos ahorran trabajo y abstraen los datos persistidos.
  • Se puede recomendar que siempre se conozca tecnologías nuevas ya que pueden ofrecer nuevas perspectivas y soluciones para problemas existentes.
  • A menos de que sea completamente necesario, no se recomienda trabajar con GitLab.

Algunos Trabajos Futuros

  • Agregar pruebas unitarias para las funcionalidades existentes.
  • Arreglar Errores en las funcionalidades existentes.
  • Aspectos Sociales para Promover Interacción entre Usuarios sobre los mismos proyectos (como Google Drive por ejemplo).
  • Llevar calificaciones automatizados, manuales e híbridas dentro de la aplicación y funcionalidades relacionados como sincronización de notas con sistemas externos.
  • Mejorar integración de Git con otros backends, uso bidireccional del mismo para la persistencia de código, suporte para características mas avanzadas de Git como Ramas y Tags.
  • Introducir nuevos backends de virtualización para la ejecución de código.
  • Versionamiento de APIs.
  • Auditar Seguridad del Sistema.
  • Mejorar manejo de metadatos como el lenguaje de programación de cada proyecto.
  • Recolectar datos para apoyar en la toma de decisiones.
  • Suportar documentación en linea como los wikis en Markdown.
